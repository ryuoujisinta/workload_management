## 1. システム概要
本システムは、従業員（ユーザー）の工数（作業時間）を管理・承認するためのWebアプリケーションである。
管理者と一般ユーザーの2つの権限が存在し、管理者が共有のタスクを作成・管理し、各ユーザー（管理者自身を含む）が日々の業務工数をタスクごとに登録・申請を行う。

## 2. 権限（ロール）とユーザー構成
- **管理者**: ユーザーとして自身の工数を登録・申請する一般機能に加え、管理者専用画面へアクセスする特権を持つ。ユーザーのアカウント発行、月次タスクの作成、全ユーザーから申請された工数の承認・却下を行う。
- **ユーザー**: 自身のアカウント管理（パスワード変更など）、該当月のタスクの選択、日々の工数登録および申請を行う。（アカウントの新規登録は行えず、管理者が登録したアカウントのみを利用する）

---

## 3. 機能要件

### 3.1 管理者向け機能
| 機能名 | 詳細 |
| --- | --- |
| **ユーザー登録機能** | 管理者が新しいユーザー（氏名、メールアドレス、初期パスワード、権限など）をシステムに登録する。 |
| **タスク登録機能** | ユーザーが工数を入力するためのタスクを「月ごと」に登録する。<br>※タスクは「プロジェクト名」と「タスク名」の情報を保持する。 |
| **タスク承認機能** | ユーザーから「登録（承認申請）」された日ごとの工数を確認し、「承認」または「却下」を行う。<br>※却下の際の理由入力は不要。<br>※承認済みの工数を編集可能にするため、管理者が「承認取り消し（差し戻し）」を行う機能を持つ。 |

### 3.2 ユーザー向け（全ユーザー共通）機能
| 機能名 | 詳細 |
| --- | --- |
| **ログイン機能** | メールアドレスとパスワードを用いて共通のログイン画面からログインする。 |
| **パスワードリセット機能** | パスワードを忘れた場合のダミー機能（将来的にSSOなどを導入予定のため、現時点ではモックデザインのみ実装）。 |
| **パスワード変更機能** | ログイン後のユーザー自身がパスワードを変更する。 |
| **タスク登録（選択）機能** | 管理者がその月用に作成したタスク一覧の中から、自分が今月担当するタスクを選択し、自身の工数入力画面に追加する。 |
| **工数登録機能** | 「タスク登録」で選択したタスクに対して、作業時間を入力する。<br>- **画面構成**: 1週間単位のカレンダー形式（前週・次週への遷移可能）<br>- **アクション**: 以下の操作を**日ごと（該当日の全タスクの工数をまとめて）**に実行可能。<br>  1. **一時保存**: 入力内容をドラフト状態として保存する。<br>  2. **登録（申請）**: 一時保存した該当日の全工数をまとめて管理者へ承認申請する。<br>  3. **登録取り消し**: 申請中の状態を取り消し、再度編集可能にする（管理者が「承認済」とした場合はユーザー側から取り消し不可）。 |

## 4. 画面一覧（UI構成案）

### 管理者画面（特権ユーザーのみ表示）
1. **管理者ダッシュボード**: 申請待ちの日別工数一覧などを表示
2. **ユーザー管理画面**: ユーザーの一覧表示・追加・編集・削除
3. **タスク管理画面**: 月ごとのタスク（プロジェクト名＋タスク名）の一覧・追加・編集・削除
4. **タスク承認画面**: ユーザーから申請された日別工数の確認、承認/却下アクション、承認取り消し

### ユーザー画面（全ユーザー共通）
1. **ログイン画面** / **（ダミー）パスワード再設定画面**
2. **マイページ（設定）**: パスワード変更
3. **マイタスク選択画面**: 当月の全体タスク一覧から自分のタスクを選択・追加
4. **工数タイムシート画面（週次）**:
   - 縦軸：選択したタスク
   - 横軸：月〜日（日付）
   - 各日付の列の下部に「一時保存」「登録（申請）」「登録取り消し」ボタンを配置（日単位でのアクション用）

---

## 5. 状態遷移（ステータス）の定義
日別の工数データ（その日の全タスク分をまとめた1日単位のデータ）は以下のステータスを持つ：
1. **未入力 / 編集中 (Draft)** : 初期状態、またはユーザーが「一時保存」した状態。
2. **申請中 (Pending)** : ユーザーが該当日の「登録」ボタンを押下し、管理者へ1日分の工数を申請した状態。ユーザーは「登録取り消し」でDraftへ戻すことが可能。
3. **承認済 (Approved)** : 管理者が「承認」した状態。ユーザー側ではデータの編集・登録取り消しが完全に不可となる（管理者が「承認取り消し」を行った場合にのみDraftやPendingに戻る）。
4. **却下 (Rejected)** : 管理者が「却下」した状態。ユーザー側へ差し戻しとなり、未入力・編集中(Draft)へ戻る。

## 6. 技術スタック（実装方針）
本システムは、UIの柔軟性と将来的な拡張性（SSO連携など）を考慮し、以下のモダンなフルスタック構成で実装を行う。

- **フレームワーク**: Next.js (App Router) + React
- **言語**: TypeScript
- **データベース**: PostgreSQL
- **ORM**: Prisma または Drizzle ORM
- **認証**: Auth.js (旧 NextAuth.js) ※初期はダミー認証/Credentials機能とし、将来的なSSO導入を想定
- **スタイリング・UI**: Tailwind CSS + shadcn/ui
